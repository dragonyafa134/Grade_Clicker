# Grade Clicker (Кликер оценок)

## Название проекта
**Grade Clicker** — интерактивное мобильное приложение, разработанное в рамках курса по разработке мобильных приложений. Пользователь набирает баллы, кликая по изображению. При достижении определённых порогов меняется отображаемая оценка, фоновое изображение и цвет интерфейса, что наглядно демонстрирует реакцию приложения на изменение данных.

## Цель работы
Изучить жизненный цикл Activity (Activity Lifecycle) в Android и научиться правильно обрабатывать переходы между состояниями приложения. Особое внимание уделено сохранению состояния при изменении конфигурации устройства (например, при повороте экрана) с помощью механизма `rememberSaveable`.

## Ответы на контрольные вопросы

**1. Какие три метода вызываются при запуске приложения?**
При запуске приложения последовательно вызываются три метода:
1. `onCreate()` — Activity создаётся, выполняется начальная настройка интерфейса.
2. `onStart()` — Activity становится видимой для пользователя.
3. `onResume()` — Activity получает фокус и становится активной (готова к взаимодействию).

**2. Чем отличается onPause() от onStop()?**
- `onPause()` вызывается, когда Activity теряет фокус, но остаётся частично видимой (например, поверх неё появляется диалоговое окно или открывается панель уведомлений). Этот метод должен выполняться быстро, так как следующая Activity не начнёт выполняться, пока не завершится `onPause()` текущей.
- `onStop()` вызывается, когда Activity становится полностью невидимой для пользователя (например, пользователь нажал кнопку «Домой» или переключился в другое приложение).

**3. Когда вызывается onRestart()?**
Метод `onRestart()` вызывается после `onStop()`, когда Activity возвращается на передний план из остановленного состояния (например, пользователь возвращается в приложение из списка последних приложений). После `onRestart()` всегда следуют вызовы `onStart()` и `onResume()`. Важно: `onCreate()` при этом повторно не вызывается, так как Activity уже была создана ранее.

**4. Что происходит при повороте экрана?**
При повороте экрана происходит изменение конфигурации устройства (configuration change). Android по умолчанию полностью уничтожает текущую Activity (вызываются `onPause()` → `onStop()` → `onDestroy()`) и затем создаёт её заново (`onCreate()` → `onStart()` → `onResume()`). Если не принять мер, все данные, не сохранённые специальным образом, будут потеряны.

**5. В чём разница между remember и rememberSaveable?**
- `remember` сохраняет значение только при рекомпозиции (обновлении интерфейса в рамках текущей Activity). При пересоздании Activity (например, после поворота экрана) значение сбрасывается.
- `rememberSaveable` сохраняет значение не только при рекомпозиции, но и при пересоздании Activity, вызванном изменением конфигурации (поворот, смена языка и т.д.). Он использует механизм сохранения состояния (`Bundle`), что позволяет восстановить данные после воссоздания Activity.

**6. Почему onCreate() должен вызывать super.onCreate()?**
`super.onCreate()` вызывает реализацию метода в родительском классе (`AppCompatActivity`). Это необходимо для корректной работы Android-системы: родительский класс выполняет важную внутреннюю инициализацию Activity. Без этого вызова приложение может работать некорректно или завершиться с ошибкой.

**7. Для чего используется класс Log?**
Класс `Log` используется для вывода отладочных сообщений в консоль Logcat. Это помогает разработчику отслеживать работу приложения, проверять последовательность вызовов методов (особенно жизненного цикла), а также диагностировать ошибки. Основные методы: `Log.d()` (debug), `Log.v()` (verbose), `Log.i()` (info), `Log.w()` (warning), `Log.e()` (error).

**8. Какой метод вызывается только один раз за время жизни Activity?**
Метод `onCreate()` вызывается только один раз за весь жизненный цикл Activity — в момент её создания. При последующих перезапусках (после `onStop`) или пересозданиях (после поворота экрана) он будет вызван снова, но каждый такой вызов соответствует новому экземпляру Activity. В рамках одного экземпляра Activity `onCreate()` выполняется единожды.
